# /learn — Distil a correction into a principle (or rule)

Run the engine in [`.docs/principles/how-to-learn.md`](../../.docs/principles/how-to-learn.md)
on the most recent correction(s) in this session (or on $ARGUMENTS if given).

Steps:
1. Identify the concrete correction (quote it).
2. Find the root cause (why), not the symptom.
3. Confirm it generalises (applies 3+ times) — else stop and say so.
4. Scan `.docs/principles/README.md` + `.docs/common-issues/` for an existing
   entry to **sharpen/merge** instead of adding a sibling.
5. FORK (principle-default): a compile/test/lint/build result settles it → RULE
   (`.docs/common-issues/` + catalog row); judgment/taste → PRINCIPLE
   (`.docs/principles/` + index row); unsure → principle.
6. Write the file in the canonical shape; keep it tight.
7. Show the exact diff (new/edited files + index/catalog rows) and STOP for review.
   Do not commit unless the user says so.
