# Recent Updates

Chronological log of what shipped / was learned in this workspace, **one file per calendar month**
(`YYYY/MM.md`). The Operating Soul scans the **current month's** file on the first message of each
session, so recent context is always in reach.

## What goes here
- A dated one-liner per meaningful change, with **why** (not just what).
- Pointers to the `.docs/principles/` or `.docs/common-issues/` entry it produced.
- Per sub-project tagging once there are several (e.g. `[proj-x]`).
