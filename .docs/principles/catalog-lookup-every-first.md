# Look up before answering — every first message
**When this applies:** the first user message of every session.
**Principle:** read what is already known before composing the first answer; over-scan rather than under-scan.
**Why:** scanning the index + one detail file is cheap; re-discovering a cataloged pattern is expensive. The old "substantive message" filter was a judgment call that misfired.
**How to apply:**
- Turn 1: scan `.docs/principles/` + `.docs/common-issues/` index against the message keywords.
- Any overlap → open the detail file before answering.
- Pure greeting / off-topic → proceed without the read; do not fabricate a match.

Enabled by [[inject-recall-upfront-method-just-in-time]] — front-loading the index at SessionStart is the
placement-strategy that makes this lookup automatic (recall present before you act), not a thing to remember.
