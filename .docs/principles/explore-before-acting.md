# Explore before acting on diffs
**When this applies:** the user asks "what's different / X?" about a diff, two configs, or a topic.
**Principle:** a question is exploration, not a work order — wait for an action verb before mutating anything.
**Why:** the user interrupted a `cp` with "i just want to know the diferent first"; they want to understand the landscape before deciding what (if anything) to change.
**How to apply:**
- Answer with description + impact only.
- A "?" on a topic name = "tell me more", not "do something".
- Run file-mutating commands only after an explicit verb ("copy", "sync", "do it", "yes").
